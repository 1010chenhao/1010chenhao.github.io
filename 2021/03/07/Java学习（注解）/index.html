注解Annotation

1. 定义

最大的作用：配合反射用于代替配置文件！！！

注解为我们在代码中添加信息提供一种形式化的方法，使我们可以在源码、编译时、运行时非常方便的使用这些数据。

Java内置注解：

  • @Override：表示当前类中的方法将覆盖父类的方法。如果方法名称和父类中的不同，则会报错

  • @Deprecated：表示被标注的方法已经被废弃了

  • @SuppressWarnings：关闭不当的编译器警告信息。除非你确定编译器的警告信息是错误的，否则最好不要使用这个注解。

2.元注解

元注解用于对注解的注释，说明注解的作用域、运行阶段、继承关系等。

元注解 功能 参数 例
@Target 表示注解作用于什么地方，如类、方法、成员变量 TYPE, METHOD, FIELD @Target(ElementType.TYPE)
@Retention 表示在什么级别保存注解信息，一般使用RUNTIME（把注解属性保存在内存中，多用于反射） SOURCE, CLASS, RUNTIME @Retention(RetentionPolicy.RUNTIME)
@Documented 将此注解包含在JavaDoc中
@Inherited 表示允许子类继承父类的注解
