接口的继承

接口的继承和类继承不一样，接口完全支持多继承，即一个接口可以有多个直接父接口。和类继承相似，子接口扩展某个多接口，将会获得父接口中定义的所有抽象方法、常量。

一个接口继承多个父接口时，多个父接口排在extends关键字之后。

接口和抽象类

接口和抽象类都不能被实例化，它们都位于继承树的顶端，用于被其他类实现和继承。抽象类和接口都包含抽象方法，实现接口或继承抽象类的普通子类都必须实现这些抽象方法。

接口体现的是一种规范，规定了实现者必须向外提供哪些服务，制定了系统各模块应该遵循的标准。而抽象类作为系统中多个子类的共同父类，可以被当成系统实现过程的中间产品，这个中间产品已经实现了系统的部分功能，还需要进一步完善。

如果使用static来修饰一个内部类，则这个内部类属于外部类本身，而不属于外部类的某个对象。

局部内部类

如果把一个内部类放在方法里定义，则这个内部类就是局部内部类，局部内部类仅在该方法里有效。

匿名内部类

匿名内部类适合创建那种只需要一次使用的类，创建匿名内部类时会立即创建一个该类的实例，这个类定义后立即消失。匿名内部类不得重复使用。

匿名内部类一般用于重写接口或抽象类中的抽象方法，但又不需要重复使用。
