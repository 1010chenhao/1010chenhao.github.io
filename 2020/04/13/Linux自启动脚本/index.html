关于Linux自启动脚本

原理：

Linux系统开机会执行以下几个步骤：

  • 读取/etc/inittab配置文件，设定系统运行级别
  • 系统初始化
  • 执行/etc/rc.d下的rc脚本，其中/etc/rc.d/rc.local自启动脚本留给用户自定义功能

方法三：使用systemd文件

从 Ubuntu 16.10 开始的新版本全部使用 systemd 实现初始化时服务的自启动

systemd 默认读取 /etc/systemd/system 下的配置文件，该目录下的文件会链接/lib/systemd/system/下的文件

所以，我们要在/etc/systemd/system中创建service脚本文件：

一般正常的启动文件主要分成三部分

[Unit] 段: 启动顺序与依赖关系
[Service] 段: 启动行为,如何启动，启动类型
[Install] 段: 定义如何安装这个配置文件，即怎样做到开机启动

方法四：使用crontab命令

  1. 编写脚本文件script.sh并赋予可执行权限
  2. crontab -e编辑crontab文件
  3. @reoot 绝对路径/script.sh

在一些 Linux 发行版（如 ArchLinux），默认情况下 cron 功能是被关闭的，
使用如下命令，将对应服务的开启即可

sudo systemctl enable cronie
